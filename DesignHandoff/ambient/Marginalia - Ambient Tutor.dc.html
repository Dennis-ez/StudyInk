Inkling Marginalia · Reimagined Ambient Tutor

The tutor you barely have to talk to.

You loved Marginalia — so let's take its idea to the end of the road. The original still waited to be asked: arm a tool, draw a lasso, type a question. That's three actions before a single answer. This version inverts it. The tutor reads along as you write, keeps its thoughts in the margin, and offers exactly the right help on the exact line — so the common case costs one tap, often zero.

The shift: from pull (you summon a panel) to push, politely (it leaves a mark you can ignore). The best input is no input; the second best is a tap on something already on screen.

01
Offered, not requested

Help appears as a quiet margin mark the moment it's useful. You never open anything to discover the tutor noticed — it already told you, in ink you can ignore.

02
A tap, never a form

No arming, no lasso, no empty text field staring back. Tap the thing; the tutor figures out what you meant and leads with the few actions that actually fit it. Typing is the fallback.

03
Already thinking

It reads continuously on-device and pre-computes the likely help, so answers feel instant. You rarely see a spinner — the thought was ready before you tapped.

The look

The margin is the tutor's lane

One idea carries the whole redesign: the red margin rule becomes where the tutor lives. At rest it's empty. When the tutor has something, a tiny amber glyph settles onto the exact line — a check, a gentle correction, a "stuck?" — aligned to your handwriting like a teacher's marks. Tap a glyph and a note unfolds from it. Nothing ever lands on top of your work uninvited.

A few you didn't ask for

Small touches that make the ambient model feel considered rather than creepy.

Fix-it in your hand

"Fix it for me" rewrites the line in a handwriting that matches yours, just tinted amber — so corrections still look like your page.

Exam-aware

If you're clearly self-testing, it stops autocompleting and only checks after — practice stays honest.

Speaks your language

Matches the language you're writing in (incl. Hebrew/RTL) and your level of formality — math stays an LTR island.

The lane is a map

Scroll back and the margin glyphs show exactly where you struggled — a heat-map of a study session, for free.

One mark at a time

Never more than one open margin note; new help waits as a glyph so the page is never crowded.

Earns the interruption

If you dismiss the same kind of nudge twice, it backs off that category for the session. It learns your tolerance.

Engineering

Build handoff — Ambient Tutor

Everything below extends the shipped Foolscap token contract and the existing SwiftUI surfaces — it does not invent a parallel system. Where it replaces or adds to a component, the original name is given. Targets: iPad Pro 12.9″ landscape, PencilKit canvas, all four skins × light/dark, LTR + RTL.

reads as: token → geometry → glyph system → intelligence → motion → per-surface redline → data model → SwiftUI map → a11y/perf/rtl → build order → acceptance.

All four skins already define --secondary / --success / --ai-circle; the ambient layer only adds the lane geometry, ghost, and the behavioural constants above. Nothing here is hard-coded per screen.

2

The margin lane — geometry

laneA non-scrolling overlay column pinned to the leading edge of the canvas, width --lane-width, height = page. It is a logical inline-start inset, so it moves to the right edge in RTL with zero extra code.
at restEmpty & transparent. When the tutor is actively reading, the lane gets --lane-tint (a 7% wash that fades the red rule warmer) — the only "it's on" cue.
glyph yEach glyph's vertical center = the baseline-center of the line/strokes it refers to (snapped to the canvas content, not the screen — it scrolls/zooms with the page). Horizontal center = lane center.
densityMax one open note at a time; additional items remain glyphs. If >5 glyphs are visible, collapse runs of ✓ into a single "✓ all clear ↑" chip.
note originA tapped glyph unfolds a MarginNoteView (the redesigned bubble) anchored to that glyph, growing toward content (inline-end). Same 300px card, amber glow, tone strip = the glyph's category color.
z--z-ai-marks for glyphs & ghost (above ink, below toolbar), --z-bubble for the open note. Never above modals/toasts.

4

Intelligence & orchestration

The loop
  1. Observe. On stroke-end (debounced 600ms idle), recognize the dirty region → text/LaTeX via on-device Vision/PencilKit + math OCR.
  2. Segment. Group strokes into lines/steps; diff against last snapshot so only new content is evaluated.
  3. Evaluate. Each segment → {type, isCorrect?, confidence, value, suggestion, ghostNext}. Cheap heuristics first; model call only for ambiguous/high-value.
  4. Gate. Emit a MarginItem only if conf ≥ --conf-mark AND value passes the sensitivity threshold (§politeness).
  5. Pre-compute. Cache the full answer for any emitted glyph so tap → note is instant (no spinner).
Sensitivity → thresholds
Offno glyphs, no ghost; tap-to-ask only
Subtleerrors only · conf ≥ .90 · no ghost · no stuck-?
Helpfulerrors + ghost + stuck-? + notes · conf ≥ .82
Focusall suppressed; queue a post-test summary

Adaptive: two dismissals of a category → drop its threshold weight for the session (the "earns the interruption" rule).

Effort/exam detection: if segments arrive in steady cadence with no backtracking and the page is titled like a test (or Focus is on), suppress ghost + autocomplete and switch to check-after-the-fact. Practice integrity over helpfulness.

Data model (sketch)

struct Anchor { // where, in page-content space (scrolls/zooms with canvas)
  var strokeIDs: [PKStrokeID]; var rect: CGRect; var lineIndex: Int
}
enum Glyph { case correct, attend, hint, note }
enum Tone  { case teaching, correct, correction, error }
struct MarginItem: Identifiable {
  let id: UUID; var anchor: Anchor; var glyph: Glyph; var tone: Tone
  var confidence: Double; var value: Double
  var answer: RichText?      // pre-computed → instant on tap
  var fixInk: PKDrawing?     // "Fix it for me" payload, your-style amber
  var state: ItemState       // .glyph .open .acted .dismissed .resolved
}
struct GhostSuggestion { var anchor: Anchor; var latex: String; var confidence: Double }
struct ReviewCard { var prompt: RichText; var answer: RichText; var due: Date; var sourceNote: NoteID }
// aiStore (transient): items:[MarginItem], ghost:GhostSuggestion?, openItem:UUID?,
//                       warmAnchor:Anchor?, sensitivity:.off/.subtle/.helpful/.focus

Margin items are transient (recomputed from ink, not persisted) except ReviewCards and a per-note dismissed-categories set. AI ink (fix-it, accepted ghost, answer-in-ink) is stored as tagged PKStrokes with an aiGenerated attribute so export-strip and one-tap-undo work.

Accessibility · RTL · performance

Accessibility
  • Glyph tap target ≥ 44pt (24 glyph + padding)
  • VoiceOver names the glyph + tone: "Correction on line 3, double-tap to view"
  • Each glyph is a focusable a11y element ordered by line
  • Notes announce tone first ("Correction:…"); math read as LaTeX
  • Color never sole signal — shape differs per glyph (works in Graphite + colorblind)
RTL (Hebrew)
  • Lane is inline-start → moves to the right edge automatically
  • Notes unfold toward inline-end (leftward); tone strip mirrors
  • Ghost text flows RTL; math & code stay LTR islands
  • Voice + verb chips mirror; "flick to accept" direction flips
Performance
  • Over a live PencilKit canvas — glyphs are shape layers, no blur
  • One open note max → ≤1 material/blur layer on screen
  • Reader debounced 600ms; diff so only new strokes evaluate
  • Pre-computed answers cached → tap is instant, no spinner
  • Ghost is a text layer until accepted; lane redraws only on scroll/zoom/new item
